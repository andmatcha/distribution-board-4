# distribution-board-2

## コマンド

- アップロード
```
pio run -t upload
```

- アップロード(ポート指定)
```
pio run -t upload --upload-port ポート名
```

- printfの出力を表示
```
./stlink_monitor.zsh
```

- 接続を確認
```
st-info --probe
```

## 使用パーツ

- STM32F103TBU6 (マイコン)
- SN65HVD75DGKR (半二重 RS-485 トランシーバ)
- AMT21 (エンコーダー)
- MCP2562FD-E/P (CANトランシーバ)

## CAN通信仕様

### 受信CAN ID一覧

| CAN ID | 用途                   | 説明                                                |
| ------ | ---------------------- | --------------------------------------------------- |
| 0x208  | ニョッキDCモーター制御 | キーボード用(DCモーター1)とUSB用(DCモーター2)の制御 |
| 0x1FF  | 把持サーボモーター制御 | 把持のサーボモーター制御(データ配列の4,5のみ使用)   |

### 0x208: ニョッキDCモーター制御

CANデータフォーマット (DLC=5以上推奨):

| Byte | 機能                                | 値  | 動作                                     |
| ---- | ----------------------------------- | --- | ---------------------------------------- |
| 0    | DCモーター1<br>(キーボードニョッキ) | 0   | 停止                                     |
|      |                                     | 1   | 出して引く動作<br>(正転1秒→逆転1秒→停止) |
| 3    | DCモーター2<br>正転制御             | 0   | 停止                                     |
|      |                                     | 1   | 正転 (出っ張る)                          |
| 4    | DCモーター2<br>逆転制御             | 0   | 停止                                     |
|      |                                     | 1   | 逆転 (引っ込む)                          |

### 0x1FF: 把持サーボモーター制御

CANデータフォーマット (DLC=6以上):

| Byte | 機能         | 説明                                         |
| ---- | ------------ | -------------------------------------------- |
| 4,5  | サーボ制御値 | 16ビット符号付き整数<br>(ビッグエンディアン) |

**制御値の意味:**
- `data < 0`: 把持開く
- `data == 0`: 把持停止
- `data > 0`: 把持閉じる

**データ例:**
```c
// 把持を開く場合
rx_data[4] = 0xFF;  // 上位バイト
rx_data[5] = 0xFF;  // 下位バイト (data = -1)

// 把持を停止する場合
rx_data[4] = 0x00;
rx_data[5] = 0x00;  // data = 0

// 把持を閉じる場合
rx_data[4] = 0x00;
rx_data[5] = 0x01;  // data = 1
```
